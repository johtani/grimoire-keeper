Issue の実装計画に基づいてブランチを作成し、実装を行い、変更点をまとめます。

## 前提

- `/plan-issue` で実装計画が Issue にコメント済みであること
- `main` ブランチが最新であること

## 手順

1. **Issue と計画を読み込む**
   - `gh issue view <番号> --comments` で Issue 本文と計画コメントを取得する
   - 実装ステップ・変更対象ファイル・テスト方針を把握する

2. **ブランチを作成する**
   - ブランチ名: `feat/issue-<番号>-<短い説明>` or `fix/issue-<番号>-<短い説明>`
   - `git checkout main && git pull origin main && git checkout -b <ブランチ名>`

3. **実装する**
   - 計画に従って変更対象ファイルを修正する
   - テスト方針に従ってテストを追加・修正する
   - スコープを計画の内容に厳密に限定する (計画外の改善・リファクタは行わない)

4. **変更点をまとめて説明する**
   以下の形式でユーザーに報告する:
   ```
   ## 実装完了: <Issue タイトル>

   ### 変更したファイル
   - `path/to/file.py` — 変更内容の概要

   ### 実装内容
   - 実施した変更の説明

   ### 次のステップ
   `/ship` を実行してコミット・PR を作成してください。
   ```

## 注意事項
- 計画のスコープを超えた変更は行わない
- 実装後のコミット・PR 作成は行わない — それは `/ship` の役割
- テストは全ユニットテストが対象 (`uv run pytest apps/api/tests/unit/ -v`)
