GitHub Issue を分析して実装計画を立て、作業準備を整えます。

## 手順

1. **Issue を読み込む**
   - 引数が URL の場合: `gh issue view <URL>`
   - 引数が番号の場合: `gh issue view <番号>`
   - Issue のタイトル・本文・ラベル・コメントを把握する

2. **コードベースを調査する**
   - Issue の内容に関連するファイルを特定する
   - 影響範囲を把握する (サービス / リポジトリ / モデル / ルーター)
   - 既存のテストファイルを確認する

3. **実装計画を提示する**
   以下の形式で計画をまとめてユーザーに承認を求める:
   ```
   ## 実装計画: <Issue タイトル>

   ### 変更対象ファイル
   - `path/to/file.py` — 変更内容の概要

   ### 実装ステップ
   1. ...
   2. ...

   ### テスト方針
   - 追加・修正するテストの概要

   ### 懸念点 (あれば)
   - ...
   ```

4. **ユーザーの承認を得てから Issue にコメントを投稿する**
   - `gh issue comment <番号> --body "<計画の内容>"` で実装計画を Issue にコメントとして追加する
   - コメント投稿をもってこのスキルは完了

## 注意事項
- 計画を提示した後、必ずユーザーの確認を待ってから Issue にコメントを投稿すること
- 実装・ブランチ作成は行わない — 計画とコメント投稿のみ
- スコープを Issue の内容に厳密に限定する
- 実装を開始する場合は `/start-issue <番号>` を使う
